Anforderungen: Tic-Tac-Toe

Basisbereich 
--> lauffähiges Minimalspiel:

B1 – Grundlegende Darstellung
	• 3x3-Spielfeld wird dargestellt 
	• Darstellung darf zunächst erfolgen

B2 – User Controls 
	• Jedes Feld ist ein eigenes UserControl
	• Feld speichert Zustand (leer, X, O)
	• Darstellung erfolgt im Control selbst

B3 – Einfache Benutzerinteraktion
	• Spielfeld reagiert auf Klicks
	• Klick setzt Symbol (X oder O)
	• Wechsel zwischen Spielern funktioniert
	
B4 – Feldbelegung
	• Bereits belegte Felder können nicht überschrieben werden
	
B5 – Einfache Spiellogik
	• Zwei Spieler (X/O)
aktueller Spieler wird verwaltet

Aufbaubereich
--> Ideen für funktionale Erweiterungen
	
E1 – Gewinnprüfung 
	• Reihen, Spalten, Diagonalen werden geprüft
	• Spiel endet bei Gewinn
	
E2 – Spielsteuerung 
	• Reset-Funktion
	• Spielfeld wird geleert

Anspruchsvolle Erweiterungen 
--> Ideen für coole Features

A1 – Gewinnanzeige
	• grafische Hervorhebung (Linie etc.)
	• Gewinnmeldung
	• Unentschieden wird ebenfalls erkannt
	
A2 – Design
	• Farben, Layout, Skalierung
	
A3 – Statistik
	• Sieg-Zähler (X / O)

A4 – Umpositionieren
	• Man kann max. 3 Elemente positionieren
Danach nur noch umpositionieren
